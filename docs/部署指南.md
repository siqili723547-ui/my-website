# GitHub Pages 部署指南

本指南将教你如何将个人作品集网站部署到 GitHub Pages，完全免费！

## 什么是 GitHub Pages？

GitHub Pages 是 GitHub 提供的静态网站托管服务，特点：
- ✅ 完全免费
- ✅ 自动 HTTPS
- ✅ 全球 CDN 加速
- ✅ 支持自定义域名
- ✅ 与 GitHub 仓库无缝集成

## 部署步骤

### 第一步：创建 GitHub 账号

如果还没有账号，访问 [github.com](https://github.com) 注册。

### 第二步：创建新仓库

1. 点击右上角 `+` → `New repository`
2. 填写仓库信息：
   - **Repository name**: `my-website`（或你喜欢的名字）
   - **Public**: 选择 Public（必须公开才能免费使用 Pages）
   - **Add a README**: 可选勾选
3. 点击 `Create repository`

### 第三步：上传代码

#### 方法 A：通过网页上传（适合初学者）

1. 在仓库页面点击 `uploading an existing file`
2. 将 `my-website/src/` 文件夹内的所有文件拖拽上传
3. 在文件名中添加 `.gitignore` 和 `README.md`（如果有的话）
4. 填写 commit 信息：`Initial commit`
5. 点击 `Commit changes`

#### 方法 B：通过 Git 命令行（推荐学习）

```bash
# 进入项目目录
cd my-website

# 初始化 Git 仓库
git init

# 添加所有文件
git add .

# 提交
git commit -m "Initial commit"

# 关联远程仓库（替换 YOUR_USERNAME）
git remote add origin https://github.com/YOUR_USERNAME/my-website.git

# 推送到 GitHub
git branch -M main
git push -u origin main
```

### 第四步：启用 GitHub Pages

1. 进入仓库页面
2. 点击 `Settings` 标签
3. 左侧菜单找到 `Pages`
4. 在 `Build and deployment` 部分设置：
   - **Source**: 选择 `Deploy from a branch`
   - **Branch**: 选择 `main` 分支，文件夹选择 `/root`
5. 点击 `Save`

### 第五步：等待部署

- 部署需要 1-2 分钟
- 点击刷新等待出现绿色对勾 ✓
- 你的网站地址：`https://YOUR_USERNAME.github.io/my-website/`

## 自定义内容

### 修改个人信息

编辑 `src/js/main.js` 中的配置：

```javascript
const siteConfig = {
    name: '你的名字',
    subtitle: '你的简介',
    about: '详细介绍...',
    skills: ['技能1', '技能2', '技能3'],
    social: [
        { name: 'GitHub', url: '你的GitHub链接', icon: 'gh' },
        { name: '邮箱', url: 'mailto:你的邮箱', icon: '✉' }
    ]
};
```

### 修改项目列表

编辑 `content/projects.json` 文件：

```json
{
    "projects": [
        {
            "title": "项目名称",
            "description": "项目描述",
            "tags": ["标签1", "标签2"],
            "link": "项目链接",
            "github": "GitHub链接"
        }
    ]
}
```

### 修改文章列表

编辑 `content/articles.json` 文件。

### 修改配色方案

编辑 `src/css/style.css` 中的 `:root` 变量：

```css
:root {
    --primary-gradient: linear-gradient(135deg, #667eea 0%, #764ba2 100%);
    /* 更改这里的颜色值 */
}
```

## 常见问题

### Q: 网站显示 404？

A: 检查以下几点：
1. 仓库必须是 Public
2. 等待几分钟让部署完成
3. 确认 URL 正确：`username.github.io/仓库名/`

### Q: 更新内容后网站没变化？

A: GitHub Pages 有缓存，等待 5-10 分钟或强制刷新浏览器（Ctrl+F5）。

### Q: 如何使用自定义域名？

A:
1. 在仓库根目录创建 `CNAME` 文件，内容为你的域名
2. 在域名提供商处添加 DNS 记录
3. 详见 GitHub Pages 官方文档

### Q: 如何添加图片？

A:
1. 将图片放入 `src/assets/images/` 文件夹
2. 在代码中引用：`assets/images/图片名.jpg`

## 本地预览

在修改代码时，可以在本地预览效果：

### 方法 1：直接打开
双击 `src/index.html` 文件

### 方法 2：使用本地服务器（推荐）

```bash
# Python
cd src
python -m http.server 8000

# 或使用 Node.js
npx serve src

# 然后访问 http://localhost:8000
```
